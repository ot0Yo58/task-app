# Long Term Memory for Task AI

## 1. このファイルの役割

このファイルは、豊晴さん専用タスクAIが長期的に参照する記憶です。

profile.md は「固定プロフィール・判断ルール」を書く場所です。  
memory.md は「使っていく中で分かってきた傾向・進行状況・学習内容」を蓄積する場所です。

## 3. 作業傾向

- 大きなタスクより、小さく分解された具体的な一手があると動きやすい。
- 「完全版コード」「貼り替え可能なコード」「具体的なコマンド」があると進めやすい。
- エラーが出たときは、原因説明だけでなく、次に何を実行すればいいかが重要。
- GitHub、Vercel、GAS、Python、Ollamaなど複数の技術が絡むときは、順番に整理すると進みやすい。
- 知識欲が強く、深掘りし始めると作業が広がりやすい。
- そのため、今日やることは3件程度に絞るとよい。

---

## 4. タスク管理上の注意

- 期限未設定タスクが増えると、優先順位が曖昧になりやすい。
- 家族・健康・お金・仕事に関わるタスクは放置リスクを高めに見る。
- 開発系タスクは「完成させる」ではなく、「1ファイル作る」「1つの機能だけ試す」などに分けるとよい。
- AIはタスクを勝手に完了・削除しない。
- タスクの追加・編集・完了・削除は、必ず確認を挟む。

---

## 5. AI連携の次の目標

- task_ai.py で profile.md と memory.md を読み込む。
- GASから取得したタスク一覧と一緒にOllamaへ渡す。
- Ollamaに「豊晴さん向け」に今日やるべきタスクを提案させる。
- 将来的には、会話や作業結果から memory.md への追記候補を作らせる。
- ただし、AIが勝手にmemory.mdを書き換えるのではなく、最初は確認後に追記する。
